# 1\.0\.0 参考项目技术文档

# CrushPilot MVP 开源项目复用方案

## 最终技术方案

```Plain Text
前端：保留现有 CrushPilot Chat 页面
后端：Python + FastAPI
工作流：LangGraph
模型调用：LangChain Core
上下文持久化：LangGraph Checkpointer
知识库：本地 Markdown / YAML / JSON
开发数据库：SQLite
线上数据库：PostgreSQL
通信方式：HTTP + SSE
```

MVP 不使用多 Agent、ReAct、MCP、向量数据库、Langfuse、Redis、语音和微信 Bot。

## 代码复用原则

```Plain Text
1. 不完整 Fork 任何大型 Agent 项目。
2. 只复制结构清晰、依赖较少的模块。
3. 复制代码时保留原项目 License 和版权声明。
4. 业务代码统一重新命名为 CrushPilot。
5. 禁止把第三方仓库直接作为运行时依赖。
6. 第三方 Skill 只作为知识来源，不执行其中脚本。
7. 所有迁移代码必须通过现有测试和类型检查。
8. 每次只迁移 MVP 当前需要的能力。
```

## 第一批 Skill

只实现以下四个 Skill：

```Plain Text
reply-suggestion
reply-rewrite
cold-recovery
date-invitation
```

每个 Skill 文件必须包含：

```YAML
id: reply-suggestion
name: 回复建议
description: 根据对方消息和最近聊天上下文生成回复
trigger_examples:
  - 这句话怎么回
  - 她说今天很累
  - 我应该回复什么
required_context:
  - latest_message
  - recent_messages
retrieval_tags:
  - reply
  - natural
output_requirements:
  - recommended_reply
  - alternatives
  - judgement
constraints:
  - 不油腻
  - 不连续追问
  - 不情绪勒索
  - 不过度推进
```

## 知识卡片格式

知识库统一使用 YAML 或 Markdown Front Matter。

```YAML
id: invitation-low-pressure-001
scene: date-invitation
stage:
  - just_met
  - early_chat
tags:
  - low-pressure
  - common-interest
  - specific-plan
principle: 邀约应尽量延续双方已经聊过的话题。
positive_example: 你不是说想去那家店吗，周六下午一起去试试？
negative_example: 你周末能不能出来和我约会？
reason: 前者有明确由头，心理压力更低。
risk_level: low
```

检索条件：

```Plain Text
Skill
用户意图
关系阶段
关键词
用户风格要求
最近一条对方消息
```

第一版使用关键词和标签评分，不使用 Embedding。

## 上下文管理

LangGraph State：

```Python
class CrushChatState(TypedDict):
    messages: Annotated[list[AnyMessage], add_messages]
    thread_id: str
    user_id: str | None
    current_skill: str | None
    relationship_stage: str | None
    user_goal: str | None
    style_preferences: list[str]
    retrieved_knowledge: list[dict]
    last_generated_replies: list[str]
    conversation_summary: str | None
    final_response: dict | None
```

上下文规则：

```Plain Text
保留最近 12 条完整消息
超出部分生成摘要
保留最近一次生成结果
保留用户明确提出的风格要求
不自动建立复杂用户画像
不自动推断永久关系档案
```

开发环境：

```Plain Text
SQLite Checkpointer
```

线上环境：

```Plain Text
PostgreSQL Checkpointer
```

## API

```Plain Text
POST /api/v1/chat
GET /api/v1/threads
GET /api/v1/threads/{thread_id}
DELETE /api/v1/threads/{thread_id}
GET /health
```

核心请求：

```JSON
{
  "thread_id": "thread_xxx",
  "message": "她说刚下班很累，我怎么回？"
}
```

核心结果：

```JSON
{
  "skill": "reply-suggestion",
  "judgement": "对方正在表达疲惫，适合先接住情绪。",
  "recommended_reply": "那今天先放过你，早点回去休息。",
  "alternatives": [
    "听起来今天挺累的，回去早点躺平。",
    "辛苦了，今晚适合什么都不干。"
  ],
  "warning": null
}
```

前端使用 SSE 接收流式内容。
