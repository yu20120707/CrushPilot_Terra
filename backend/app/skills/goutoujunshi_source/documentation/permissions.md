# 权限边界

## 运行角色

| 角色 | 能力 | 不能做什么 |
| --- | --- | --- |
| 用户 | 提供信息、选择是否采用建议、决定是否启用宿主工具 | 不通过本 Skill 自动获得他人账号、位置或私密数据 |
| Codex代理 | 读取 Skill 与按需参考文档，在任务中生成建议 | 不自动发送消息、不代表用户作出法律或医疗决定、不绕过工具批准 |
| 仓库维护者 | 修改公开文件、审查贡献和发布版本 | 无权访问使用者在私人Codex任务中的内容 |
| 贡献者 | 通过Issue或PR提交公开内容 | 不应提交可识别的真实关系案例或未经授权的私密材料 |

## 资源与操作矩阵

| 资源 | 用户 | Codex代理 | 维护者／贡献者 |
| --- | --- | --- | --- |
| `SKILL.md`与参考资料 | 可读取 | 可读取 | 通过Git修改公开副本 |
| 当前任务对话 | 可提供和查看 | 仅按宿主上下文访问 | 默认不可访问 |
| 用户本地文件 | 按宿主授权 | 仅在宿主提供工具且用户授权时访问 | 不可访问 |
| 外部消息或账号 | 用户自行操作 | 本 Skill 不要求写权限 | 不可访问 |
| GitHub仓库 | 公开读取 | 无运行时要求 | 按GitHub仓库权限贡献 |

## 数据与权限实现

仓库没有数据库、令牌、角色声明或行级安全机制。运行权限完全由宿主Codex环境和其工具批准系统决定。Skill正文只能约束建议方式，不能提升代理权限，也不能替代程序级访问控制。
