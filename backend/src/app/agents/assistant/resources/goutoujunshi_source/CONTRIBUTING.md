# 参与贡献

感谢帮助狗头军师变得更准确、更包容、更能解决真实问题。

## 适合贡献的内容

- 修正文献引用、事实、法律口径或过时内容。
- 补充不同性取向、性别身份、文化、年龄、残障或家庭结构下的关系场景。
- 改进首次建档、多人对象管理、情绪支持和决策流程。
- 增加更自然的中文表达、地区化语气或可执行对话模板。
- 提交能够暴露读心、刻板印象、过度承诺、控制或安全风险的测试场景。
- 修复断链、目录、元数据和验证脚本问题。

## 知识贡献标准

提交知识文档时，请明确区分：

1. 较强证据：系统综述、元分析、纵向研究、正式统计或现行法律文件。
2. 理论框架：有学术来源但不应直接当作个体诊断的模型。
3. 流行说法：MBTI配对、依恋标签、爱情语言及网络恋爱术语等。
4. 经验策略：可能实用，但效果依赖场景且缺乏稳定研究支持的话术或方法。

优先引用原始研究、政府、法院、国际组织和大学资料。不要用单一自媒体文章支撑医学、法律或普遍因果结论。

## 内容边界

不接受帮助实施性胁迫、下药、偷拍、跟踪、勒索、财产诈骗、冒充身份、散布隐私、规避明确拒绝或制造系统性依赖的内容。可以提交这些机制的风险分析和低风险替代方案。

请避免：

- 把人格类型、性别或性取向写成命运。
- 把心理疾病标签用于羞辱或未经诊断的读心。
- 把任何关系形式默认成更高尚或更低级。
- 使用无法核实的“必胜”“百分之百”“所有人都”式结论。

## 提交流程

1. Fork 仓库并创建主题分支。
2. 只修改与本次贡献相关的文件。
3. 运行 `python3 scripts/validate_skill.py`。
4. 在 Pull Request 中说明修改内容、原因、证据来源和用户影响。
5. 若修改建议流程，至少提供一个成功场景和一个失败／边界场景。

贡献内容需要遵守本项目的许可证和行为准则。
