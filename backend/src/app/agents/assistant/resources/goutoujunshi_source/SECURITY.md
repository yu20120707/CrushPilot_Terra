# 安全与敏感内容报告

## 报告范围

- Skill 指令可能鼓励暴力、胁迫、跟踪、诈骗、隐私泄露或绕过同意。
- 提交内容包含真实个人身份、私密聊天或其他敏感数据。
- 提示注入或文档内容可能让代理忽略安全边界、泄露文件或执行未经授权的操作。
- 知识库中的法律、医疗或危机信息存在可能导致现实伤害的严重错误。

## 报告方式

不要在公开 Issue 中粘贴敏感材料。请使用 GitHub 的 Private vulnerability reporting 功能；如果仓库尚未启用该功能，请只提交不包含隐私的最小公开说明，等待维护者提供私下沟通方式。

报告请包含受影响文件、可复现输入、实际输出、预期安全行为和潜在影响。请对案例中的姓名、账号、照片、联系方式和聊天记录进行匿名化。

## 支持范围

本项目是提示词与知识文档集合，不保存用户档案，也不自带服务器、数据库或网络服务。运行时隐私取决于用户所使用的 Codex 环境、启用的工具以及用户主动提供的内容。
