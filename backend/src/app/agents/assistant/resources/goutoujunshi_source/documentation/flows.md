# 关键流程

## 1. 首次使用与建档

**参与者**：用户、Codex代理
**前置条件**：当前任务中不存在完整用户档案
**成功结果**：形成最小必要档案，并确定眼前问题和情绪强度

1. 用户显式调用 Skill 或提出符合描述的恋爱问题。
2. 代理先处理紧急情绪或必须马上回复的事项。
3. 代理收集本人MBTI和主观综合评分。
4. 代理为一个或多个目标对象收集代号、MBTI、主观评分和关系阶段。
5. 代理了解关键事件、联系频率、双方投入、主要障碍、目标和情绪强度。
6. 代理复述档案，标记未知与矛盾，不填造缺失事实。

**隐私边界**：只需使用代号；不需要真实姓名、地址、账号、身份证件或私密照片。

## 2. 常规关系分析

**前置条件**：拥有足够的用户叙述或档案
**成功结果**：用户获得可解释、可执行且保留选择权的建议

1. 用简短语言确认情绪与内在冲突。
2. 把叙述拆分为已知事实、合理推测和关键未知。
3. 根据场景读取必要参考文档。
4. 比较互惠、可靠、吸引、价值观、现实可行性、风险和机会成本。
5. 给出明确首选建议和关键理由。
6. 必要时提供稳健、策略或强势版本。
7. 收束为一件小事、观察窗口、停止条件和反馈信号。

**拒绝路径**：若用户要求实施违法、强迫、跟踪、诈骗或绕过同意的行为，代理拒绝该操作部分，解释风险并提供低风险替代。

## 3. 聊天材料分析与即时回复

1. 判断素材来自聊天截图、导出文本、复制对话、用户转述、通话、线下互动或混合来源。
2. 只把可确认的原文、发送者、顺序、间隔和行为写入事实；把含义解释放入推测。
3. 先处理用户当下情绪，再判断本轮主要目标是承接、调侃、推进、邀约、澄清还是收线。
4. 用户只要一句回复时，第一屏先给一条符合用户口吻、可直接发送的首选成品。
5. 按对方积极、含糊或拒绝反应给出后续分支；需要练习时每轮只模拟一条回复并等待用户接话。

## 4. 主动约会与第一次见面

1. 没有明确拒绝或不适时，给出至少一个主动联系、具体邀约或真实表达动作。
2. 第一次见面建议保持简洁，覆盖可交谈的地点、共同体验、具体欣赏和结束时的真实反馈。
3. 根据双方对恐怖、密闭、运动、噪音和人群的偏好选择活动，不把“吊桥效应”写成保证吸引的公式。
4. 肢体互动结合关系与现场适当创造自然、低强度、可退出的机会，并根据对方反应调整。
5. 一次含糊不等于永久退出；没有明确拒绝时可在合理间隔后换时间或场景再邀请一次。

## 5. 多目标对象比较

1. 为每个对象保留独立代号与事件记录。
2. 不用一个对象的MBTI、行为或话术填补另一个对象的信息。
3. 分别评估互惠、可靠、相处体验、现实可得性和长期风险。
4. 比较“高评分但不可得”和“评分较低但互惠稳定”的真实差异。
5. 保留“暂不选择任何人”作为合法选项。

## 6. 高情绪强度

1. 先缩小时间范围，避免连发、报复、突然删除或其他不可逆动作。
2. 建议暂存消息、离开刺激源或联系可信的人陪伴。
3. 在情绪下降后再进入事实分析和策略选择。
4. 若存在立即自伤或伤人危险，转入紧急支持，不继续只给恋爱话术。

## 7. 暴力、控制与危机

1. 识别暴力、性胁迫、跟踪、财务控制、隐私威胁或诈骗信号。
2. 读取法律安全与危机转介文档。
3. 优先人身安全、可信支持、证据保存和当地专业服务。
4. 不建议用户在不安全环境中单独摊牌或以伴侣咨询替代安全计划。
5. Skill只提供一般信息；现实紧急情况由当地紧急服务和合格专业人员处理。
