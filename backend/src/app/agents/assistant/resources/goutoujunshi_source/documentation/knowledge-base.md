# 知识库治理

## 分层

| 层级 | 目录 | 用途 |
| --- | --- | --- |
| 行为规则 | `SKILL.md` | 决定何时询问、怎样分析、怎样输出和何时切换安全流程 |
| 核心知识 | `references/knowledge/` | 研究、理论、法律、社会、人文与哲学背景 |
| 实用材料 | `references/practical/` | 沟通结构、情绪回应、拒绝、赞美、主动权和表达脚手架 |

经典社交课程只按能力原子转译：机制、证据和风险放在 `references/knowledge/`，伦理化练习与输出结构放在 `references/practical/`。课程原文、逐字稿、媒体、版本清单和研究工作区不进入运行时 Skill。

## 证据等级

1. **较强证据**：元分析、系统综述、大型纵向研究、正式统计和现行法律文件。
2. **理论框架**：可用于提出问题，但不能直接诊断个体或预测结局。
3. **流行说法**：可帮助理解用户语言，必须明确证据局限。
4. **经验策略**：以适用条件、预期效果、暴露风险和长期代价呈现。

## 包容性要求

- 不把异性恋、婚姻、单偶或生育设为默认终点。
- 分开处理性取向、性别身份与关系结构，不混为同一概念。
- 不根据群体身份推断性格、忠诚、性行为或关系能力。
- 对所有关系形式使用同一核心标准：知情同意、互惠、尊重、自主、可靠和修复能力。
- 当地区法律对不同人群保护不一致时，明确区分伦理建议与法律现实。

## 更新规则

- 法律、人口、政策、求助渠道和数字平台信息引用前复核日期与地区。
- 新增研究时优先补充原始或权威来源，不只添加二手总结。
- 修改实用话术时保留“适用场景—效果—代价—下一步”结构。
- 新增知识文档时同步必需文件清单或最低数量，不锁死精确总数。
- 删除内容时检查`SKILL.md`路由和所有相对链接。
- 不把真实用户的私密案例直接沉淀进公开知识库。
- 默认按当前问题读取 1–3 份参考；需要证据或风险校准时再增量加载，不批量载入全库。
