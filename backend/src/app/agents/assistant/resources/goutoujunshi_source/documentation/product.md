# 产品定位

## 产品主张

狗头军师不是替用户做主的“感情裁判”，而是一个同时处理情绪、信息和行动的恋爱决策助手。它帮助用户从混乱叙述、聊天截图或导出文本中建立关系档案，识别事实和读心之间的距离，比较继续、主动推进、沟通、观察、降级投入或退出的真实代价，并把建议转化为可发送的话和可执行的下一步。

## 核心用户与价值表达

| 用户 | 典型困境 | 价值表达 |
| --- | --- | --- |
| 暧昧与约会中的用户 | 看不懂信号，反复猜测，不知道怎样推进 | 把零散互动整理成互惠证据，并给出邀约、观察窗口和停止条件 |
| 提供聊天记录的用户 | 截图、导出文本、转述和线下互动混在一起，担心被过度脑补 | 区分素材来源、可见原文、事实、推测和未知，再按上下文生成可发送回复 |
| 冲突中的伴侣 | 情绪很强，想连发、质问、冷战或分手 | 先降低冲动成本，再提供能表达感受、边界和具体请求的对话方案 |
| 多人选择中的用户 | 心动、条件、稳定与现实可行性相互冲突 | 分开维护对象档案，比较可靠、互惠、价值观、现实和长期选择权 |
| 分手或背叛后的用户 | 想复合、报复、自责或无法停止关注 | 区分止痛冲动与关系可修复证据，设计恢复、验证或体面退出路径 |
| LGBTQ+与多元关系用户 | 普通建议默认异性恋、性别分工或单一婚姻脚本 | 不预设性别角色，以知情同意、互惠、现实条件和个人目标为判断基础 |
| 内容创作者与关系教育者 | 需要结构化框架，而不是碎片化金句 | 提供跨心理、社会、法律、人文、哲学与性学的主题知识和使用边界 |

## 差异化能力

1. **情绪与决策一体化**：不是分析完就结束，而是确认用户在当下能否执行。
2. **多人档案**：原生支持一个用户对应多个目标对象，不把事件和判断串线。
3. **行为证据优先**：人格类型只生成问题，不直接生成结论。
4. **聊天材料识别**：区分截图、导出文本、转述、通话、线下互动和混合素材，不补写材料外的情节。
5. **即时话术与演练**：先给一条可发送的首选成品，再提供反馈分支或按需模拟对话。
6. **主动约会支持**：帮助用户表达真实喜欢、准备第一次见面、设计共同体验，并根据反馈调整自然接触。
7. **策略分级**：既不把所有社交策略妖魔化，也不隐藏短期效果背后的长期代价。
8. **跨学科路由**：按当前场景读取必要知识，不一次塞入整套资料。
9. **明确停止条件**：把“再看看”变成可观察、可截止、可退出的行动计划。

## 非目标

- 不诊断心理疾病，不替代心理治疗或医疗服务。
- 不充当律师、警方、紧急救援或家暴风险评估机构。
- 不预测特定对象一定会爱上用户。
- 不帮助绕过同意、拒绝、隐私或法律边界。
- 不用MBTI、性别、性取向或星座决定关系命运。
- 不保存跨任务的用户私密档案；持久化能力取决于未来明确设计和用户同意。

## 产品成功信号

当前仓库没有遥测。未来评估应使用匿名或合成案例，关注：

- 首次使用是否完整收集最小必要档案，而不过度审问。
- 回答是否准确区分事实、推测和未知。
- 聊天截图、导出文本、转述和线下互动是否被正确识别且没有跨媒介脑补。
- 是否给出明确建议、理由、下一步和停止条件。
- 用户只要回复话术时，第一屏是否有一条符合原文和用户口吻的可发送成品。
- 普通推进场景是否避免反复劝退，并在第一次见面、表达喜欢和再次邀约上提供具体行动。
- 是否避免把用户引向更高风险或更少选择权的行为。
- 不同性取向、性别身份和关系结构下，建议质量是否一致。
- 高风险场景是否正确切换到安全优先流程。

## 路线图

- 建立匿名合成案例评测集，覆盖约会、冲突、多人选择、分手和危机场景。
- 为引用资料加入更新时间、来源类型和复核状态元数据。
- 增加可选的结构化关系档案模板和用户可控的导入／删除机制。
- 扩展地区化法律与求助信息，并将其与通用关系建议严格分离。
- 增加繁体中文和英文内容，同时保留本地文化与家庭语境。
