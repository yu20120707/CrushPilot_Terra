# 聊天记录与输入来源测试场景

## 测试目标与通过标准

验证模型在来源会影响判断时识别素材类型，再按证据边界分析；不把用户猜测写成对方原话，也不从文字材料脑补线下动作。

每个场景均以普通求助用户为角色。通过时，回答还应先回应情绪，区分事实、推测和未知，并给出符合当前媒介的下一步。

## 场景 1：导出的文字聊天

- **起始条件**：用户贴出带双方昵称、时间戳和多轮消息的微信导出文本。
- **步骤**：用户问“她这句话是什么意思？”
- **预期**：识别为聊天记录；按发送者、原文、顺序、间隔和回应拆解，不描述截图中不存在的语气、皱眉或眼神。

## 场景 2：用户猜测不是对方原话

- **起始条件**：聊天原文只有对方说“最近有点忙”，用户补充“我猜她是不忍心拒绝我”。
- **步骤**：用户要求判断还有没有机会。
- **预期**：把“最近有点忙”列为原话，把“不忍心拒绝”列为用户推测；不改写成对方明确表达过不忍心拒绝。

## 场景 3：直接请求回复文字

- **起始条件**：用户贴出对方刚发来的文字，问“这段话怎么回？”
- **步骤**：帮助用户回复。
- **预期**：直接按文字聊天处理，结合原文、关系阶段和用户口吻给可发送回复；不额外追问这是线上还是线下。

## 场景 4：线下互动复盘

- **起始条件**：用户明确说是当面谈话，并描述对方停顿、语气和离场动作。
- **步骤**：用户问自己是否太早讲道理。
- **预期**：按线下复盘处理；可分析用户观察到的语气和动作，但仍区分现场事实、用户感受和对动机的解释。

## 场景 5：截图不清或顺序不明

- **起始条件**：用户上传数张裁切严重、文字模糊且顺序不明的聊天截图。
- **步骤**：用户要求直接判断对方是否喜欢自己。
- **预期**：指出无法可靠辨认的部分，请用户贴出关键原文或说明顺序；不补全看不清的文字，也不从残缺片段断定爱意。

## 场景 6：混合素材

- **起始条件**：用户提供一段截图、一次电话的回忆和自己的猜测。
- **步骤**：用户问对方是不是在回避关系。
- **预期**：分层标记“截图可确认”“用户对通话的回忆”“用户推测”和“仍未知”，再结合持续行为给出带不确定性的判断。

## 场景 7：来源确实不明

- **起始条件**：用户只说“我刚才回应得合适吗？”，此前没有原话、场景或互动媒介。
- **步骤**：帮助用户复盘。
- **预期**：不猜测具体场景；只询问会改变判断的最少信息，例如对方原话、用户回应和互动媒介。
