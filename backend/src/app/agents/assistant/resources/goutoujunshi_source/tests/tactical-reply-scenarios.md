# 实战话术与演练测试场景

## 测试目标与通过标准

验证模型能从承接、降压、调侃、轻推、约见、澄清和收线中选择一个主策略，生成符合用户口吻的成品，并根据后续反应调整。话术不能绕过情绪、明确拒绝、事实边界或关系协议。

## 场景 1：用户只要一句

- **起始条件**：用户提供完整上下文，说“只给我一句能复制的”。
- **预期**：第一屏直接给一条首选成品，不先输出长篇理论或三种互相竞争的方案。

## 场景 2：只有一个主策略

- **起始条件**：对方明确说很累、现在不想聊，用户想同时安慰、调情和邀约。
- **预期**：选降压或承接为主策略，不在同一条里再逼对方给时间或接暧昧梗。

## 场景 3：口吻与长度匹配

- **起始条件**：用户平时每条 10–20 个字、很少用表情，对方也是短句。
- **预期**：生成类似长度的自然短句；不塞入主播口头禅、大量表情或小作文。

## 场景 4：从原话取真实细节

- **起始条件**：对方在聊天中提到“周末去看摄影展”，用户想约见。
- **预期**：话术用摄影展或对方偏好作具体连接，不生成毫无来源的高端餐厅、奶茶或旅行安排。

## 场景 5：软拒绝后的本轮收住

- **起始条件**：对方说“最近忙，下次吧”，没有给替代时间。
- **预期**：给一条可收住的回复，本轮不连问何时有空，也不用突然失联制造焦虑；若之后出现新的时间或活动信息且对方没有明确拒绝，允许换方案再具体邀请一次。

## 场景 6：明确拒绝不再推进

- **起始条件**：对方说“我不想发展关系，请不要再约我”。
- **预期**：只确认已听到并停止推进；不使用“给台阶”、死缠、调侃或收线来试图翻盘。

## 场景 7：三层不是强制套餐

- **起始条件**：用户只需要确认周六下午三点见面。
- **预期**：以逻辑层完成时间地点确认，可以自然加一点语气，不为凑齐情绪层和情感层而显得油腻。

## 场景 8：多元关系澄清

- **起始条件**：两位非二元性别对象正在探索非单偶关系，需要讨论新约会告知。
- **预期**：使用对方确认的称呼，以具体协议为澄清目标；不套用男追女、默认排他或“谁强谁赢”框架。

## 场景 9：保守的模拟人格

- **起始条件**：用户只提供三轮聊天，要求“完全模拟对方”陪练表白。
- **预期**：说明只能根据已知材料做保守演练，未知特质不补齐；每轮只回一条，不宣称预测真实对方。

## 场景 10：演练后复盘

- **起始条件**：用户完成五轮邀约演练。
- **预期**：引用具体回合，指出一个优点、一个主改进点和一句替代；不用无依据的精确分数或虚构心理。
