## 用 VS Code 连接后的操作

在 VS Code Remote-SSH 终端中执行：

```bash
cd ~
git clone https://github.com/yu20120707/CrushPilot_Terra.git
cd CrushPilot_Terra/deploy
```

将本地 `deploy/.env` 上传/复制为服务器上的 `deploy/.env`，然后务必将 `POSTGRES_PASSWORD` 改成高强度随机密码。

```bash
docker compose up -d --build
docker compose ps
curl http://127.0.0.1:8080/health
```

首次部署或原始知识文件变更后，先在容器内离线派生安全卡，再构建向量索引；这一步会调用已配置的聊天模型并下载本地 embedding 模型。原始资料不会进入在线聊天提示词。

```bash
docker compose exec backend python -m app.knowledge_base build-cards
docker compose exec backend python -m app.knowledge_base build-index
```

浏览器访问：`http://43.138.155.193:8080`。

## 切换模型

编辑 `deploy/.env` 后重启后端服务：

```bash
# 使用 DeepSeek
MODEL_PROVIDER=deepseek

# 使用 Longxia
MODEL_PROVIDER=longxia

docker compose up -d --force-recreate backend
```

切换 embedding 模型或重新生成安全卡后，也要重新执行 `build-index`。

部署前，在腾讯云安全组放行 TCP `8080`。如果后续绑定域名并启用 HTTPS，放行 TCP `80`、`443`，且反向代理必须对 `/api/v1/chat` 关闭响应缓冲。

## 不要提交密钥

执行以下命令确认环境文件没有被 Git 跟踪：

```bash
git check-ignore -v deploy/.env
git status --short
```
